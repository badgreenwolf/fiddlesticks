Introduction to the stick

The stick is either an operation , a value of something, or a small numerical constant. Its 2 bytes long. Sticks are always in arrays of up to 16 long, and they point to one another.

 

The stick

Part Size Notes
GRAB 4 bits the index of what this stick grabs on to, set to 0 for not grabbing anything
Flags 4 bits

bits 1 and 0 tell the type of stick, 0 = NOP which means no stick here,1 is a value, 2 is an operation,3 is a constant

bits 2 and 3 are currently undefined, but should be set to zero for right now
TypeUnion 1 byte

if flags say OP this is an unsigned byte, which is the opcode of the operation. There can be up to 256 different operations

if flags say Constant, then this is a signed byte

if flags says Value then unsigned byte of the offset of the data in entire array

 

 

Stick Array Header (4 bytes)

 

size total array 1 byte from 8 bytes 164 bytes
number of sticks 4 bits from 1 to 16 sticks, 0 sticks arrays are not made
size of data 4 bits size is from 1 to 8
number of data 5 bits from 0 to 16
type of data 3 bits integer, floating
flags 8 bits currently unused

 

Stick Array is organized like below, and can be from 8 bytes to 164 bytes

 

Header 4 bytes see header above
Sticks 2 to 32 bytes see sticks above
Values 0 to 128 bytes all of the same size and type as defined in the header
